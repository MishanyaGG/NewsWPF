### Техническое задание (ТЗ)

#### 1. Модуль авторизации

При запуске приложения отображается окно авторизации с полями «Логин», «Пароль» (скрытый ввод) и кнопками «Вход», «Выход».

Вход выполняется только для предопределённой учётной записи: логин `editor`, пароль `news2025`.

При неверном логине или пароле выводится сообщение «Ошибка авторизации. Проверьте логин и пароль», поля очищаются.

После успешной авторизации открывается главное окно приложения, окно авторизации закрывается.

#### 2. Модуль управления новостями

Главное окно содержит таблицу со следующими колонками:

• ID:  
• Заголовок:  
• Категория:  
• Дата публикации:  
• Автор:  
• Количество просмотров:  
• Статус (опубликовано/черновик):

Над таблицей размещены кнопки: «Добавить», «Редактировать», «Удалить», «Обновить».

Добавление: при нажатии открывается диалоговое окно с полями:

• Заголовок (обязательное, 5–200 символов)  
• Текст новости (обязательное, 20–10000 символов)  
• Категория (обязательное, выпадающий список: «Политика», «Спорт», «Культура», «Технологии», «Экономика», «Происшествия»)  
• Дата публикации (обязательное, не позднее текущей даты)  
• Автор (обязательное, 5–100 символов, только буквы, пробелы)  
• Статус (обязательное, выпадающий список: «Опубликовано», «Черновик»)  
• Кнопки: «Сохранить», «Отмена».

Редактирование: двойной клик по строке или выделение строки + кнопка «Редактировать» открывает окно с предзаполненными полями (включая текст новости), изменение по тем же правилам.

Удаление: перед удалением выводится подтверждение: «Удалить новость "[Заголовок]"?», кнопки «Да», «Нет».

Валидация: при неверном вводе поле подсвечивается красной рамкой, кнопка «Сохранить» блокируется, выводится понятное сообщение об ошибке.

Вычисляемое поле «Количество просмотров»: увеличивается на 1 каждый раз, когда пользователь открывает новость для просмотра (двойной клик по строке в таблице). Начальное значение — 0.

#### 3. Модуль управления категориями

• Приложение содержит вторую вкладку (TabControl) «Категории» с таблицей:

• ID:  
• Название категории:  
• Описание:  
• Количество новостей (вычисляемое):

Реализованы CRUD-операции для категорий по аналогии с разделом «Новости».

#### 4. Нефункциональные требования

Приложение не должно завершаться аварийно при некорректном вводе данных.

Любое исключение перехватывается и выводится в пользовательском сообщении, не раскрывая системную информацию.

#### 5. Требования к интерфейсу (UI/UX)

Окна имеют минимальный размер: главное окно – 1024×600, диалоговые – 500×450 (для учёта поля «Текст новости»).

Приложение использует единую цветовую гамму (светлая или тёмная тема на выбор).
